    seajs

  • 这两个插件都是用来管理加载的 主要解决以下两个问题：
  1. 变量空间 通过封装来保证模块内的变量不会污染全局
  2. 减少页面引入的js链接的数量 便于管理
  • 这两个软件都是用来做模块化的
  • 模块化的好处
    • 提升开发效率
    • 便于后期维护
  • 常见的模块规范 cmd amd commenjs ECMAscript

  • 全局变量
  • 把变量绑定到一个对象身上

requirejs用法

  • 基本上和seajs类型 不过不同在于

  1. 它在页面上script中直接指定了主js 在页面上也就只有这一个 预加载 预解析

  2. 所有主要编写main.js中的内容

  3. 模块写法 模块就是用来返回对象的

二者的共同点

  1. 都可以配置路径
    seajs.config({
    path:’主路径’,
    })
  2. 都是预加载 但是seajs是懒解析
